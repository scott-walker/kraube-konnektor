Features

Everything Claude Code can do.
Now programmable.

Full access to Claude Code's capabilities through a typed TypeScript API. No features left behind.

⚡

Persistent SDK Sessions

Initialize once, query instantly. The SDK keeps a warm session running — no cold start on every question. See real-time init stages: importing → creating → connecting → ready.

⇄

Real-time Streaming

Watch Claude think. Stream responses word-by-word, see which tools are being used, and react to events as they happen. Built on async iterators — just for await.

💬

Multi-turn Sessions

Have conversations, not one-shots. Sessions remember context between queries. Resume later by session ID. Fork conversations to explore different paths.

⚡

Parallel Agents

Run multiple Claude instances simultaneously. Analyze code in one, run tests in another, check docs in a third. One line: claude.parallel([...]).

🔄

Recurring Tasks

Schedule Claude to run on intervals — like /loop but from your code. Monitor deployments, poll CI status, or run periodic code reviews. claude.loop('5m', ...)

🔧

MCP Servers & Subagents

Connect to Model Context Protocol servers. Define custom subagents with specific tools, models, and permissions. Full Claude Code agent ecosystem from code.

📋

Structured Output

Pass a JSON Schema, get validated JSON back. Extract data, parse codebases, generate configs — all type-safe. No string parsing, no regex, no prayers.

🛡

Permission Control

Full control over what Claude can do. plan mode for read-only analysis. acceptEdits for autonomous coding. Allowlists, denylists, tool-level control with glob patterns.

🔀

Swappable Backends

Today: CLI and SDK executors. Tomorrow: whatever Anthropic ships next. The IExecutor abstraction means your code never changes — only the transport does.

Before & After

From terminal to API

Before — manual CLI

# Run Claude, copy output, parse it,
# somehow feed it into your pipeline...

$ claude -p "find bugs" --output-format json \
    | jq '.result' \
    | your-script.sh

# No sessions. No streaming. No types.
# Good luck with error handling.

After — claude-connector

const claude = new Claude({ useSdk: true })
await claude.init()

const session = claude.session()
const bugs = await session.query('Find bugs')
const fixes = await session.query('Fix them')

// Typed. Streaming. Sessions. Error handling.
// Works in your existing Node.js app.
Architecture

Built to last

Clean layers, zero dependencies, swappable backends. When the SDK evolves, your code stays the same.

Your Code ↓ Claude (facade) ↓ ArgsBuilder → IExecutor → Claude Code ↑ ┌───────────┴────────────┐ CliExecutor SdkExecutor (spawn CLI) (persistent session)
Get Started

Three lines to your first query

Install the package, import it, and go. Works with your existing Claude Code subscription.

terminal 
$ npm install claude-connector
index.ts 
import { Claude } from 'claude-connector'

const claude = new Claude({ useSdk: true, model: 'sonnet' })
await claude.init()

// That's it. Ask anything.
const result = await claude.query('What does this project do?')
console.log(result.text)
